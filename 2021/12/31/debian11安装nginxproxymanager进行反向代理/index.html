Debian11安装Nginx Proxy Manager进行反代

发表于2021-12-31|更新于2022-02-05|杂项
|阅读量:

前言：

尽管网上的教程已经很多了，但我还是记录一下我自己的安装过程，说不定以后还能用到，说不定还可以给有需要的朋友一些帮助。 　　不得不说的是，NginxProxyManager使用确实还是很方便的，功能也十分的简洁。 NginxProxyManager的项目地址：https://github.com/jc21/nginx-proxy-manager

正文：

　　说起Debian，装Docker和Docker-compose还是很简单的，一条命令，解决。

apt update && apt install docker docker-compose -y

　　然后创建一个名为docker-compose.yml的文件， 　　可以用命令**cat>docker-compose.yml**并copy下面内容进去。

version: ‘3’
services:
app:
image: ‘jc21/nginx-proxy-manager:latest’
restart: unless-stopped
ports:
- ‘80:80’
- ‘81:81’
- ‘443:443’
volumes:
- ./data:/data
- ./letsencrypt:/etc/letsencrypt

　　其中，81是NginxProxyManager的管理面板的端口。 　　好的，启动它。

docker-compose up -d

　　然后访问你的IP:81就是后台地址了，默认的账号密码是

Email: admin@example.com Password: changeme

　　登录后可以进行更改。
